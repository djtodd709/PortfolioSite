Planet Zoo - Brachiation

Jun 2022

With the addition of Siamangs to Planet Zoo, the team felt it was important that swinging from beams was included in some form, a trademark gibbon behaviour. My main goal was creating a system that gave animation the ability to create impressive swinging motions, while ensuring players could build a wide variety of frame layouts. Here's how I did it.

Features

  • Integration with the game's existing climbing system
  • Object snapping system for creating frames
  • Brachiation animation system with opportunities for dynamic motions
  • Code controlled animations when transitioning between beams
  • Beam claiming system to limit clipping

When brachiating, Siamangs will look ahead to the end of their current beam. If a subsequent beam is found near the same position, the animal may attempt to traverse to this next beam, continuing it's brachiation animation. I implemented plenty of pathfinding logic to ensure only accessible, unobstructed beams are traveled to, and significant animation code to ensure the system worked smoothly.

A modular system where pieces can snap together to make a frame.

Brachiation integrates with the preexisting climbing system.

Development

I worked heavily with Design, Production, and Animation to ensure the system operated in a way that balanced player freedom with impressive animations, while at the same time ensuring animation time wasn't wasted on unnecessary elements.
In my technical design I pushed for modularity over the initially planned singular object. This made the feature more interesting for players while also reducing animation's workload by allowing a shorter animation to be retargeted to the various modular pieces.
As part of development I led weekly interdepartmental meetings on the feature, ensuring everything was coordinated and highlighting potential issues. This level of planning meant we were able to get double the animations we'd initially budgeted for.

What I'd Do Next Time

I'd have loved to experiment with some form of hand IK, allowing siamangs to grip the pole reliably, and allowing animations to be made a lot faster. This would allow for more varieties of beam to be created, as animation time was the limiting factor for this feature.
For a more practical change, it would have been useful to discuss the beam dimensions at an earlier stage to ensure they all synergized (e.g. having two quarter turn pieces cover the same distance as one half turn piece).
